# Pitch Deck Design

## Quick Reference

**Slide count (live pitch):** 10–15; **email leave-behind:** allow appendix. **One idea per slide.** Title = takeaway, not topic label.

| Slide | Job |
|-------|-----|
| Hook | Why this meeting matters now |
| Problem | Quantified pain, who feels it |
| Insight | Why incumbents fail / why now |
| Solution | Product story in 3 bullets + visual |
| How it works | Simple diagram, not architecture porn |
| Proof | Logos, metrics, case snippet |
| Differentiation | 2×2 or “why us vs X” |
| Business model | How they buy (align O4) |
| Team | Credibility, not bios wall |
| Ask | Next step, timeline, owners |

**2025–2026:** Many pitches are async video or live over Zoom — minimum 18pt font, high contrast, no paragraphs.

## Patterns & Decision Matrix

| Audience | Emphasis |
|----------|----------|
| CFO | ROI, risk, contract shape |
| CTO | Integration, security, roadmap |
| End users | Workflow relief, time saved |
| Procurement | Compliance, SLA, exit |

## Code Examples

**Slide title rewrite (bad → good):**

```text
BAD:  "Overview"
GOOD: "Cut support tickets 35% in 90 days"

BAD:  "Platform architecture"
GOOD: "Works with your stack: Salesforce + Okta + Snowflake"
```

**Speaker notes block (per slide):**

```text
[Slide 7 — Proof]
Say: "Acme cut manual reconciliations from 40h to 6h weekly."
Show: before/after chart (source: case study 2025-Q4)
If asked on data: defer detail to appendix slide A3.
```

**16:9 layout hint (for designers):**

```text
Safe zone: keep critical text inside 1120×630 px on 1920×1080 master
Logo: bottom-right, 24 px margin
```

## Anti-Patterns

| Mistake | Fix |
|---------|-----|
| Deck = brochure dump | Separate deep-dive PDF |
| Unreadable charts | Max 3 data series per chart |
| Fake metrics | Only claims you can evidence |
| No explicit CTA | Last slide names date + deliverable |
