## 2. Golden Ratio Applications in Character Design

### 2.1 The Golden Ratio (Phi = 1.618)

The golden ratio appears naturally in human proportions and can be used to create aesthetically pleasing character designs.

### 2.2 Body Golden Ratio Relationships

| Relationship | Ratio | Application |
|-------------|-------|-------------|
| Total height / navel height | ~1.618 | Navel is golden division of total height |
| Navel to sole / navel to top | ~1.618 | Lower body slightly longer than upper |
| Shoulder to fingertip / elbow to fingertip | ~1.618 | Arm segment golden ratio |
| Hip to sole / knee to sole | ~1.618 | Leg segment golden ratio |
| Forearm+hand / forearm | ~1.618 | Wrist marks golden division |

### 2.3 Facial Golden Ratio (Phi Mask)

The Marquardt Phi Mask defines ideal facial proportions:

| Measurement | Ratio to Face Width | Notes |
|------------|-------------------|-------|
| Face height / face width | ~1.618 | Hairline to chin vs bizygomatic width |
| Hairline to brow / brow to nose tip | ~1.618 | Upper face to mid-face |
| Nose tip to chin / lip to chin | ~1.618 | Lower face division |
| Mouth width / nose width | ~1.618 | Feature width relationship |
| Eye width / interocular distance | ~1.0 | Eyes are one eye-width apart |
| Interocular / nose width | ~1.0 | Eyes span = nose width |

### 2.4 Golden Ratio for Fantasy Characters

| Character | How to Apply Phi | How to Break Phi |
|-----------|-----------------|-----------------|
| Elf | Amplify golden ratios; make everything more Phi | Elongate vertically (face taller than 1.618x wide) |
| Goblin | Break ratios deliberately; nothing aligns to Phi | Head too large, limbs too short, face compressed |
| Orc | Partially broken; jaw extends below golden chin line | Widen everything; face wider than golden ratio |
| Dwarf | Keep upper body ratios, compress lower body | Legs short relative to torso; break height/navel ratio |
| Troll | Severely broken ratios; primitive, asymmetric | Arms way too long; head way too small |

### 2.5 Using Golden Ratio as a Design Tool

**Beautiful characters:** Follow golden ratio closely (elf, angel, noble)
**Ugly/monstrous characters:** Break golden ratio deliberately (goblin, troll, undead)
**Interesting characters:** Follow some ratios, break others (orc, dwarf, anti-hero)

## 4. Programmatic Proportion Verification

### 4.1 Proportion Check Script

```python
# Conceptual — E2 (Blender Script Agent) implements
import bpy
from mathutils import Vector

def verify_proportions(mesh_obj, target_head_count=7.5, tolerance=0.15):
    """
    Verify character proportions against target head-count system.
    
    Args:
        mesh_obj: The mesh object to check
        target_head_count: Expected head-count ratio
        tolerance: Allowed deviation (0.15 = 15%)
    
    Returns:
        dict: Proportion check results with pass/fail per landmark
    """
    # Get bounding box in world space
    bbox = [mesh_obj.matrix_world @ Vector(corner) for corner in mesh_obj.bound_box]
    
    # Calculate dimensions
    z_min = min(v.z for v in bbox)
    z_max = max(v.z for v in bbox)
    x_min = min(v.x for v in bbox)
    x_max = max(v.x for v in bbox)
    
    total_height = z_max - z_min
    total_width = x_max - x_min
    
    if total_height <= 0:
        return {'error': 'Zero height mesh'}
    
    head_unit = total_height / target_head_count
    
    results = {
        'total_height': total_height,
        'head_unit': head_unit,
        'target_head_count': target_head_count,
        'checks': {}
    }
    
    # Define expected landmark positions (as fraction of total height from bottom)
    landmarks = {
        'half_height': {
            'expected_z': z_min + (total_height * 0.50),
            'description': 'Crotch should be at half height',
            'tolerance': tolerance,
        },
        'navel': {
            'expected_z': z_min + (total_height * 0.60),
            'description': 'Navel at 60% height (3.0/7.5 from top = 4.5/7.5 from bottom)',
            'tolerance': tolerance,
        },
        'nipple_line': {
            'expected_z': z_min + (total_height * 0.733),
            'description': 'Nipple line at ~73% height',
            'tolerance': tolerance,
        },
        'chin': {
            'expected_z': z_max - head_unit,
            'description': 'Chin at one head-unit below top',
            'tolerance': tolerance,
        },
    }
    
    # Width checks
    shoulder_expected = head_unit * 2.0  # ~2 heads wide for male
    hip_expected = head_unit * 1.3
    
    results['checks']['shoulder_width'] = {
        'expected': shoulder_expected,
        'actual': total_width,
        'ratio': total_width / shoulder_expected if shoulder_expected > 0 else 0,
        'pass': abs(total_width - shoulder_expected) / shoulder_expected < tolerance
    }
    
    return results
```

### 4.2 Head-Count Measurement Script

```python
# Conceptual — E2 implements
def measure_head_count(mesh_obj):
    """
    Measure the approximate head-count of a character mesh.
    Assumes the mesh is in T-pose or A-pose.
    
    Returns:
        float: Estimated head-count
    """
    mesh = mesh_obj.data
    world_matrix = mesh_obj.matrix_world
    
    # Transform all vertices to world space
    world_verts = [world_matrix @ v.co for v in mesh.vertices]
    
    if not world_verts:
        return 0.0
    
    # Total height
    z_values = [v.z for v in world_verts]
    total_height = max(z_values) - min(z_values)
    
    # Estimate head height:
    # Find the topmost cluster of vertices (head)
    # Head vertices are typically the top 10-15% of the mesh
    z_threshold = max(z_values) - (total_height * 0.13)
    head_verts = [v for v in world_verts if v.z > z_threshold]
    
    if not head_verts:
        return 0.0
    
    head_z_min = min(v.z for v in head_verts)
    head_z_max = max(v.z for v in head_verts)
    head_height = head_z_max - head_z_min
    
    if head_height <= 0:
        return 0.0
    
    head_count = total_height / head_height
    
    return round(head_count, 1)
```

### 4.3 Proportion Comparison Script

```python
# Conceptual — E2 implements
def compare_proportions(mesh_obj, reference_type='human'):
    """
    Compare mesh proportions against a reference character type.
    
    Returns:
        dict: Deviations from reference proportions
    """
    # Reference proportion tables (normalized, total height = 1.0)
    references = {
        'human': {
            'head_count': 7.5,
            'half_height': 0.50,     # Crotch
            'shoulder_width': 0.267,  # 2.0 heads / 7.5
            'hip_width': 0.173,       # 1.3 heads / 7.5
            'arm_length': 0.40,       # Shoulder to fingertip
            'leg_length': 0.47,       # Crotch to sole
        },
        'goblin': {
            'head_count': 4.0,
            'half_height': 0.45,      # Slightly lower center
            'shoulder_width': 0.30,   # 1.2 heads / 4.0 — narrower
            'hip_width': 0.20,
            'arm_length': 0.50,       # Long relative arms
            'leg_length': 0.35,       # Short legs
        },
        'orc': {
            'head_count': 6.5,
            'half_height': 0.48,
            'shoulder_width': 0.385,  # 2.5 heads / 6.5 — very wide
            'hip_width': 0.23,
            'arm_length': 0.43,
            'leg_length': 0.42,
        },
        'elf': {
            'head_count': 8.5,
            'half_height': 0.52,      # Slightly higher center
            'shoulder_width': 0.165,  # 1.4 heads / 8.5 — narrow
            'hip_width': 0.141,
            'arm_length': 0.42,
            'leg_length': 0.50,       # Long legs
        },
        'dwarf': {
            'head_count': 4.5,
            'half_height': 0.44,      # Lower center (short legs)
            'shoulder_width': 0.444,  # 2.0 heads / 4.5 — very wide
            'hip_width': 0.30,
            'arm_length': 0.38,
            'leg_length': 0.35,       # Short legs
        },
    }
    
    ref = references.get(reference_type, references['human'])
    
    # Measure actual mesh proportions
    bbox = [mesh_obj.matrix_world @ Vector(corner) 
            for corner in mesh_obj.bound_box]
    
    total_height = max(v.z for v in bbox) - min(v.z for v in bbox)
    total_width = max(v.x for v in bbox) - min(v.x for v in bbox)
    
    if total_height <= 0:
        return {'error': 'Invalid mesh dimensions'}
    
    actual = {
        'shoulder_width': total_width / total_height,
        # Additional measurements would need vertex analysis
    }
    
    deviations = {}
    for key in actual:
        if key in ref:
            expected = ref[key]
            deviation = (actual[key] - expected) / expected
            deviations[key] = {
                'expected': expected,
                'actual': actual[key],
                'deviation_pct': round(deviation * 100, 1),
                'pass': abs(deviation) < 0.15,  # 15% tolerance
            }
    
    return deviations
```

### 4.4 Automated Proportion Report

```python
# Conceptual — E2 implements
def generate_proportion_report(mesh_obj, character_type='human'):
    """Generate a comprehensive proportion report."""
    head_count = measure_head_count(mesh_obj)
    verification = verify_proportions(mesh_obj, target_head_count=head_count)
    comparison = compare_proportions(mesh_obj, reference_type=character_type)
    
    report = {
        'character_type': character_type,
        'measured_head_count': head_count,
        'verification': verification,
        'comparison': comparison,
        'warnings': [],
    }
    
    # Check for common issues
    references = {
        'human': 7.5, 'goblin': 4.0, 'orc': 6.5,
        'elf': 8.5, 'dwarf': 4.5, 'troll': 5.5
    }
    expected = references.get(character_type, 7.5)
    
    if abs(head_count - expected) > 1.0:
        report['warnings'].append(
            f'Head count {head_count} significantly deviates from '
            f'expected {expected} for {character_type}'
        )
    
    return report
```

## 5. Common Proportion Mistakes and Fixes

### 5.1 Universal Mistakes

| Mistake | What Happens | How to Detect | How to Fix |
|---------|-------------|---------------|-----------|
| Legs too short | Character looks childish or dwarfish | Half-height point above crotch | Extend leg metaball elements; check half-height = crotch |
| Arms too short | Looks awkward, T-rex arms | Fingertips don't reach mid-thigh | Extend arm chain; wrists should align with crotch |
| Head too small | Looks like a mannequin | Head count > 9 unintentionally | Increase head metaball radius |
| Head too large | Looks like a child or bobblehead | Head count < 6 unintentionally | Decrease head metaball radius |
| Hands too small | Looks weak, unfinished | Hand < 0.6 head lengths | Scale hand elements to ~0.75 head length |
| Feet too small | Looks unstable, about to fall | Foot < 0.8 head lengths | Scale foot to ~1.0 head length |
| Shoulders too narrow | Looks weak even for lean characters | Shoulder width < 1.5 heads (male) | Widen shoulder metaball spacing |
| Navel too high/low | Torso proportions look wrong | Check against 3.0 head position | Adjust torso segment lengths |
| Elbows too high/low | Arms look wrong when at sides | Elbows should be at navel/waist | Adjust upper arm length |

### 5.2 Creature-Specific Mistakes

| Creature | Common Mistake | Fix |
|----------|---------------|-----|
| Goblin | Head not big enough; looks like a small human | Push head to 1/3.5 of height; exaggerate ears and nose |
| Goblin | Too muscular; goblins should be wiry | Reduce all limb radii; emphasize joints over muscle |
| Orc | Neck too long; breaks the "no neck" silhouette | Merge head into shoulders; trapezius should reach skull |
| Orc | Jaw not prominent enough; looks like a green human | Extend mandible, add tusk protrusions, widen jaw |
| Elf | Not tall/slender enough; looks like a human with pointy ears | Add 1-2 head-counts; narrow everything by 15% |
| Elf | Ears too large; looks like a goblin | Keep ears elegant and small; 1.5-2x human, not 2.5x |
| Dwarf | Looks like a scaled-down human | Widen shoulders dramatically; barrel the chest |
| Dwarf | Legs too long; loses the stocky feel | Legs should be max 35% of total height |
| Troll | Arms not long enough | Arms should reach to or below knees |
| Troll | Head too big for body | Small head is key to the "dumb brute" read |

### Quick Proportion Checks (3-Second Verification)

1. **Crotch at half-height?** If not, legs or torso are wrong
2. **Elbows at navel?** If not, arm proportions are off
3. **Wrists at crotch?** If not, forearm length is wrong
4. **Fingertips at mid-thigh?** If not, hand or arm length is off
5. **Shoulders ~2 heads wide (male)?** If not, too narrow or wide
6. **Foot ~1 head long?** Most commonly undersized
7. **Hand = chin to mid-forehead?** Most commonly undersized
