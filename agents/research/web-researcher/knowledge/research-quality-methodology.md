# Research Quality Methodology

## Source Selection and Credibility

### Credibility Tiers
1. **Tier 1 — Primary sources**: Original research, official documentation, direct data
2. **Tier 2 — Expert synthesis**: Peer-reviewed journals, established analyst firms (Gartner, Forrester)
3. **Tier 3 — Quality journalism**: Major tech/industry publications with editorial standards
4. **Tier 4 — Community sources**: Use for signals only, verify claims independently

### Red Flags for Low-Quality Sources
- No author attribution
- No date or "last updated" information
- No citations for factual claims
- Promotional tone disguised as objective content
- Contradicts multiple Tier 1-2 sources without explanation

## Evidence Synthesis Framework

When multiple sources address the same question:

1. **Identify consensus** — What do 3+ independent sources agree on?
2. **Note conflicts** — Where do credible sources disagree? Why?
3. **Weight recency** — More recent data overrides older for fast-moving domains
4. **Cite depth** — Group supporting sources together, don't cite each point individually

### Citation Format Best Practice
Group sources supporting the same claim:
> "Adoption of X has increased significantly [(Source A, 2024); (Source B, 2024)]"

Not:
> "Source A says X increased. Source B also says X increased."

## Research Workflow

1. **Frame the question precisely** — vague questions produce vague answers
   - Bad: "Is X technology good?"
   - Good: "What are the adoption rates and failure modes of X in enterprise settings?"

2. **Search strategy**
   - Start with academic/authoritative queries
   - Technical language produces better results than layman terms
   - Example: Translate "Does cold make you sick?" → "Does cold temperature exposure increase risk of upper respiratory infection?"

3. **Triangulate findings** — verify key claims with at least 2 independent sources

4. **Synthesize, don't summarize**
   - Don't: list what each source said
   - Do: integrate findings into a coherent answer with citations

## Output Structure

For research deliverables:
1. **Introduction** — restate the question and scope
2. **Evidence** — grouped by theme, not by source
3. **Conclusion** — direct answer with confidence level
4. **Caveats** — what's uncertain or contested
5. **Sources** — with access date for web sources

## Fact-Checking Checklist

- [ ] Claim supported by primary or Tier 1/2 source?
- [ ] Data is recent (within 2 years for fast-moving fields)?
- [ ] No single-source dependency for critical claims?
- [ ] Potential bias of source acknowledged?
- [ ] Quantitative claims include sample size/methodology?
