# Validation Checklists (Universal)

Every final level proposal/review must include a Validation section and explicitly pass these checks (or list exceptions).

## Universal level validation

### Design intent
- [ ] Main intention is clear.
- [ ] Teaches/tests/twists one primary idea.
- [ ] Expected player emotion is defined.
- [ ] Fits current progression (no unexpected spike without reason).

### Player knowledge
- [ ] Required knowledge listed.
- [ ] New knowledge is isolated before being tested.
- [ ] No untaught mechanic is mandatory.
- [ ] Dependency model has no impossible links.

### Spatial logic
- [ ] No unsupported objects.
- [ ] No invalid surface placements.
- [ ] Goals are reachable and readable.
- [ ] Traversal objects connect meaningful spaces.
- [ ] Hazard boundaries are readable.
- [ ] Decor does not imply false affordance.

### Gameplay logic
- [ ] Intended path is physically/mechanically possible.
- [ ] Obvious wrong path handled (fails readably).
- [ ] Failure teaches something.
- [ ] Shortcut/bypass risk acceptable or blocked.
- [ ] Every gameplay object has a purpose (purpose audit).

### Readability
- [ ] Goal is visible or clearly implied.
- [ ] First action is readable.
- [ ] Main focal point is clear.
- [ ] Feedback explains state changes.
- [ ] No hidden information required.

### Difficulty
- [ ] Attempt budget matches tier (tutorial/early/mid/late/boss).
- [ ] Complexity does not spike unexpectedly.
- [ ] Breather levels exist after high-intensity stretches.
- [ ] Mastery path is optional.

### Production / feasibility
- [ ] Uses implemented mechanics only, unless flagged as new requirement.
- [ ] New art/code/audio needs listed as handoff notes (not implemented by D14).
- [ ] Game-pack loaded (if exists) OR temporary Design Brief written (if pack missing).
- [ ] Game-pack constraints respected (if present).
